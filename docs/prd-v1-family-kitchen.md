# 家庭菜谱管理工具 V1 PRD

## Summary
- 产品定位：移动优先 Web/PWA 的家庭饮食管理工具，服务一个家庭内多成员协作，核心目标是“记录今天吃了什么、家里还有什么、接下来做什么”。
- 首版优先级：家庭日常记录 > 菜谱沉淀 > AI 辅助，强调低维护成本、清晰责任归属、温暖家庭化 UI。
- 成功标准：新家庭 10 分钟内完成建家与首批数据录入；单次餐食记录控制在 30-60 秒；AI 推荐能基于库存、历史和偏好给出可执行建议。

## Key Changes
- 家庭与账号
  - 顶层对象为“家庭”，采用邀请制账号体系；家庭创建者创建家庭并邀请成员加入。
  - 角色仅保留 `Owner` 与 `Member`；`Owner` 管理成员、家庭设置与 AI 配额，`Member` 可新增/编辑食物、菜谱、库存与餐食记录。
  - 所有新增、编辑、删除操作必须记录操作者、时间、对象类型和摘要，用于“谁做了什么”的追踪与活动流展示。
- 食物与菜谱
  - 建立统一 `Food` 概念，类型固定为 `自做菜`、`外卖`、`外出就餐`、`成品食品`。
  - `Food` 支持名称、分类、口味标签、来源信息、就餐场景、图片、备注、收藏状态；仅 `自做菜` 强绑定 `Recipe`，其他类型可选填来源商家/餐厅/品牌。
  - `Recipe` 包含份量、准备时长、难度、原料清单、步骤、技巧、图片、适用场景，并允许从食材库引用原料。
  - 菜谱支持“复做”与“变体记录”，但 V1 不做复杂版本分叉，只保留最近编辑人与编辑时间。
- 食材与库存
  - `Ingredient` 作为基础食材词条，包含名称、分类、常用单位、默认存放位置、图片。
  - `InventoryItem` 采用轻量库存模型：数量、单位、状态、购买日期、可选保质/到期日期、备注、存放位置；V1 不做跨单位自动换算。
  - 支持低库存和临期提醒，并可一键加入购物清单；购物清单为 V1 正式模块，不做预算与支付能力。
  - 当记录一餐使用了带菜谱的自做菜时，系统只给出“建议扣减库存”，由用户确认，不做无提示自动扣减。
- 每日饮食记录
  - `MealLog` 按日期和餐别记录，餐别固定为 `早餐`、`午餐`、`晚餐`、`加餐/夜宵`。
  - 每条记录可关联多个 `Food`，支持照片、参与就餐成员、备注、心情/满意度标签。
  - 提供“快速记录”入口：最近吃过、收藏菜、拍照后补信息、复制昨日餐食。
  - 首页以“今天”为中心展示：今日已吃、待处理提醒、库存预警、AI 推荐、家庭动态。
- AI 与图片
  - AI 能力限定为实用助手型：对某个菜/菜谱提问、按库存推荐今天可做的菜、按现有食材生成菜谱草稿、解释临期/缺货风险、基于拍照辅助生成记录草稿。
  - AI 作用域固定为 `单菜问答`、`家庭库存问答`、`今日吃什么推荐`、`菜谱生成/改写`，避免做成开放式泛聊天。
  - 图片能力为“上传为主，AI 生成为辅”：`Food` 与 `Ingredient` 都支持上传图片；`Food` 额外支持 AI 生成封面图；食材图允许上传或使用 AI/默认图库占位。
  - 图片上传需支持手机拍照、相册选择、压缩与封面选择；V1 不做复杂 OCR/冰箱自动识别。
- UI 与体验
  - 视觉风格定义为温暖、家庭化、照片驱动、移动优先；使用大卡片、大点击区、家庭成员头像和轻提示，而不是企业后台式表格界面。
  - 一级导航固定为 `首页`、`食物`、`菜谱`、`食材`、`记录`、`AI`、`我的家庭`。
  - V1 必含搜索、筛选、收藏、最近使用、空状态引导、首次建家引导。
  - 二期再考虑营养摄入、卡路里、外部外卖平台同步、语音录入、儿童/长辈简化模式。

## Public Interfaces / Core Types
- 核心实体
  - `Family`, `User`, `Membership`, `ActivityLog`
  - `Food`, `Recipe`, `RecipeIngredient`
  - `Ingredient`, `InventoryItem`, `ShoppingListItem`
  - `MealLog`, `MealLogFood`, `PhotoAsset`
  - `AIConversation`, `AIRecommendation`
- 关键业务约束
  - 一个用户在 V1 只归属一个家庭，避免跨家庭权限复杂度。
  - 每个对象都必须带 `createdBy`, `updatedBy`, `createdAt`, `updatedAt`。
  - `Food.type` 为强枚举，不允许自由扩展；`MealLog.mealType` 同样为强枚举。
  - AI 接口必须显式接收上下文范围，不能在无家庭上下文下返回库存或推荐结果。

## Test Plan
- 家庭创建者可创建家庭并邀请 2 名成员加入，成员登录后新增内容时，活动流能正确显示操作者与时间。
- 可分别创建 `自做菜`、`外卖`、`外出就餐`、`成品食品` 4 类食物，且只有自做菜要求填写完整菜谱。
- 能为一个菜谱录入原料、步骤、图片，并从食材库选择至少 3 个食材形成引用关系。
- 能录入至少 5 个库存食材，包含购买日期和数量，并触发低库存或临期提醒。
- 用户可记录一顿包含多道菜的晚餐，上传至少 1 张照片，选择共同就餐成员，并在记录后收到库存扣减建议。
- AI 能回答“这个菜怎么做得更清淡”“今晚用现有食材做什么”“哪些食材快过期了”三类问题，且答案基于当前家庭上下文。
- 无上传图片时，菜品可生成 AI 封面图；已有图片时，AI 生成功能不覆盖用户原图，只作为附加选项。
- 首页在移动端首屏必须能完成 3 个高频动作：快速记一餐、看库存提醒、看 AI 推荐。

## Assumptions
- 首版默认语言为简体中文，服务对象为中国家庭使用场景。
- 登录方式按邀请制账号设计，底层可使用手机号或邮箱实现，但首版原型先使用本地演示账号与成员切换。
- V1 以“家用易坚持”为第一原则，因此库存、推荐和 AI 都优先降低录入负担，而不是追求绝对精细。
- 通知先做站内提醒与首页提示，推送通知可作为后续增强项。
- AI 结果均为辅助建议，不提供医疗或营养诊断承诺。
