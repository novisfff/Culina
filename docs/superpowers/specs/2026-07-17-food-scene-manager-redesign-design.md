# 食物场景管理弹窗重设计规格

日期：2026-07-17

状态：视觉、结构与交互已确认，待书面规格复核

## 目标

把当前占用空间大、操作按钮重复抢占视觉注意力的“场景管理”弹窗，改成图片主导但仍适合管理任务的紧凑横向卡片列表。用户应能快速识别场景、进入编辑或创建推荐场景，同时把低频且危险的删除操作收进次级入口。

本次只重做食物工作区的场景管理弹窗，不改变场景编辑/新建表单、场景数据模型、接口契约、图片生成流程或推荐场景转自定义场景的既有语义。

## 已确认方向

采用 A1“横向图片卡”方案：图片、场景信息与操作在同一行；自定义场景常驻“编辑”，删除收进“更多”菜单；推荐场景显示“创建”。

不采用文字叠图的沉浸封面卡，避免真实图片复杂时对比度不稳定；不采用双列图片格，避免长标题、描述和管理操作在手机端过度拥挤。

## 弹层结构

继续复用 `WorkspaceOverlayFrame` 与 `WorkspaceModal`，保留现有遮罩、焦点管理、Escape、backdrop 与 busy 关闭保护。

- header 保留拖拽条、eyebrow“食物场景”、标题“场景管理”、一句说明和圆形关闭按钮。
- body 只保留一个紧凑工具栏和一个场景列表，不再重复展示大段“整理入口和封面”的说明。
- 工具栏左侧显示“{N} 个场景”和短说明“整理常用入口”，右侧显示 primary“新建场景”。
- “新建场景”不再在常规手机宽度占满整行；只有极窄宽度无法保证文案与 44px 触控目标时才允许自然换行并占满可用宽度。
- modal body 是唯一滚动容器；header 与工具栏保持可见，场景列表单独占据剩余内容空间。

## 场景卡片

列表始终使用单列横向卡片，不在桌面改成网格。卡片使用项目既有暖白 surface、柔和边框、中等圆角和轻阴影，不新增色板或重阴影。

每张卡片包含：

- 左侧 4:3 场景封面。手机基准约 96×76px，平板和桌面可小幅增大，但不超过信息区高度。
- 中间信息区显示场景名称、类型标签和可选说明。
- 名称单行截断；类型标签紧邻名称，分别显示“自定义”或“推荐”。
- 说明最多两行；没有说明时不渲染占位空间。
- 右侧操作区使用不少于 44×44px 的触控目标。

自定义场景显示常驻次级“编辑”和图标式“更多”按钮。推荐场景不显示“更多”，只显示“创建”，并继续通过现有 `onOpenEditScene` 分支打开预填的新建表单。

图片加载失败或没有封面时继续复用 `MediaWithPlaceholder`，图片本身使用空 `alt`，因为相邻文字已经完整表达场景名称。

## 删除交互

当前实现直接渲染“删除”按钮并立即调用删除回调；重设计后新增显式的两步删除保护：

1. 点击自定义场景的“更多”按钮，打开锚定于该卡片的轻量菜单。
2. 菜单只包含危险项“删除场景”。
3. 点击后关闭菜单并打开项目现有 `ConfirmDialog`，标题为“删除「{场景名}」？”，说明删除后该入口将从场景列表移除，主按钮使用危险语义“删除场景”。
4. 只有确认后才调用 `onDeleteScene(scene.id)`。

菜单必须支持键盘打开与选择、Escape 关闭、点击外部关闭，并用 `aria-haspopup="menu"`、`aria-expanded` 和包含场景名的 `aria-label` 表达状态。任一时刻只允许一个场景菜单打开。

删除提交期间确认弹窗保持忙碌状态，禁止重复确认、关闭 manager、点击 backdrop 或通过 Escape 关闭。现有 `onDeleteScene` 返回 `void`，因此 View 不通过 Promise 返回值猜测结果：确认后等待目标 scene id 从新的 `sceneCards` 中消失，再关闭确认弹窗并保留 manager；若 mutation 失败且目标仍在列表中，沿用现有错误表面并保留确认上下文。

## 状态与边界

- `isUpdatingScene` 为真时，关闭按钮、新建、编辑、创建、更多菜单与删除确认全部禁用。
- 打开编辑/创建表单、关闭 manager、目标场景从列表消失时，清理已打开的菜单和待删除目标。
- 列表为空时显示简洁空状态“还没有场景”，补充说明“新建一个常用场景，快速整理食物”，并提供“新建场景”入口；工具栏仍保留主入口，但空状态按钮可作为就近操作。
- 长名称、长说明和窄屏不能挤压图片或操作区；文字区域使用 `min-width: 0` 和明确的截断规则。
- 场景数量变化后直接使用 `sceneCards.length` 更新工具栏，不新增本地派生计数。
- 待删除目标只保存最小 `{ id, name }` 快照；用新的 `sceneCards` 校验目标是否仍存在，不把完整服务端对象复制进本地状态。

## 响应式规则

### 手机 375–767px

- 使用现有 bottom sheet，宽度 100%，最大高度和安全区继续遵循 overlay 规范。
- 卡片为单列横向布局，图片基准 96×76px；信息区优先获得剩余宽度。
- 工具栏保持同一行；在 375px 仍保证“新建场景”按钮和关闭按钮触控区不少于 44px。
- 卡片垂直间距收紧，避免一屏只能看到一到两个场景。

### 平板和桌面 768px 以上

- 使用现有居中 modal，保持适合管理任务的中等宽度，不扩展为全屏工作区。
- 卡片仍为单列横向列表，封面可提升到约 112×84px。
- 工具栏、卡片内容和操作保持同一视觉轴线，不引入第二列场景网格。

## 测试与验收

### 组件测试

- manager 打开时显示紧凑工具栏、数量与新建入口。
- 自定义场景显示“编辑”和“更多”，不直接显示“删除”。
- 推荐场景显示“创建”，不显示“更多”。
- 更多菜单一次只打开一个，支持 Escape 与点击外部关闭。
- 删除菜单先打开 `ConfirmDialog`；取消不调用回调，确认只调用一次并传递正确 scene id。
- busy 时关闭、新建、编辑、创建、更多和确认删除全部不可操作。
- 空列表、无描述、长标题和长描述结构正确。
- 图片继续使用响应式 `srcSet`、`sizes` 与 decorative `alt`。
