# 餐食规划 Workflow

## 简单推荐流程

1. 读取可用库存。
2. 读取临期食材。
3. 结合最近餐食避免重复。
4. 返回 2-3 个建议。
5. 不生成草稿。

## 正式计划流程

1. 读取库存、临期、最近餐食、食物和菜谱。
2. 判断天数、餐别和约束。
3. 生成完整 `meal_plan` 草稿。
4. 校验草稿结构。
5. 调用 `meal_plan.create_draft`。
6. 返回待确认草稿。

## 修改已有计划流程

1. 从 conversation artifacts 找到 `meal_plan` 草稿。
2. 校验用户要修改的是该草稿。
3. 生成完整替换版草稿。
4. 返回新的待确认草稿。
