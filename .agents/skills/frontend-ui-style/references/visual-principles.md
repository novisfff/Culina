# Culina 产品视觉与内容原则

在判断产品气质、内容优先级、文案语气或现有页面是否发生视觉漂移时读取本文件。色彩、字体和空间层级读取 `visual-system.md`；组件、首页与 AI 模式读取 `component-patterns.md`。

## 产品气质

- 把界面理解为家庭日常工具，不是营销展示、企业后台或科技大屏。
- 优先清楚、稳定、低维护，再追求装饰效果。
- 使用温暖、家庭化、照片驱动的视觉语言；可以有轻微治愈感，但不儿童化。
- 默认使用简体中文，避免“赋能、革新、全新体验”等营销表达。

## 判断顺序

- 先判断用户正在完成什么任务，再判断当前数据和操作状态，最后决定视觉表达。
- 先保证对象、数量、日期、家庭归属和下一步真实，再考虑装饰、动效或品牌感。
- 页面主任务、主操作和恢复路径必须比推荐、解释、统计和装饰更容易找到。
- 现有页面可以提供实现事实，但不能为全橙按钮、卡片套卡片、虚假成功态或桌面压缩式手机布局背书。

## 内容与状态真实性

- success 只表示已经发生并可确认的成功；推荐、计划、草稿和等待审批不属于成功。
- loading、empty、error、disabled、partial success、cancelled 和 stale data 都要与真实状态一致。
- 错误说明发生了什么、保留了什么、用户下一步能做什么；危险确认说明对象和影响范围。
- AI 在用户确认前使用“草稿、建议、待确认”等措辞，不写成已经入库、扣减或安排完成。

## 文案

- 使用短动词和具体对象，例如“开始做”“确认草稿”“补食材”“查看记录”。
- 错误文案说明发生了什么、哪些输入被保留、用户下一步能做什么。
- 状态不能只靠颜色；图标需要文字或可访问名称支持。
- 避免医疗诊断、营养治疗承诺、绝对化建议和过度拟人。

## 禁止模式

- 通用 SaaS 后台、冷灰密集表格、蓝色主按钮体系。
- 营销落地页 hero、大面积 AI 紫渐变、霓虹发光、强玻璃拟态。
- 硬黑边框、厚重阴影、随机圆角、每个区域一套颜色。
- 抽象营销插画或假截图装饰替代真实产品信息。
- 为了“高级感”降低对比度、缩小触控区或隐藏必要状态。
