# Culina 固定响应式、弹层与可用性规范

本文件固定手机、平板、桌面、弹窗、抽屉、表单、触控和键盘的基础规格。业务内容可以在固定设备层级内增加布局断点，但不得建立第二套 mobile/tablet/desktop 定义。

## 设备层级

```css
/* Mobile */
@media (max-width: 767px) { }

/* Tablet */
@media (min-width: 768px) and (max-width: 1023px) { }

/* Desktop */
@media (min-width: 1024px) { }
```

- 手机端是独立体验，不是桌面 JSX 的隐藏或缩放版本。
- 平板端必须同时适配横竖屏，不直接复用 360px 宽的手机卡片或弹窗。
- 桌面、平板、手机共享 data、actions、model 和业务规则，可以使用不同 view、排序、导航和操作入口。
- 内容型布局可以在设备层级内部增加断点，但必须写明具体内容为何在该宽度重排。

## 页面布局

- 手机主内容宽度固定 `min(100%, 480px)` 并居中；全宽工作区、图片查看器和烹饪模式可由业务明确例外。
- 桌面内容最大宽度固定 `1440px`；页面主区块 gap 固定 24px。
- 平板页面主区块 gap 固定 20px；手机固定 16px。
- 手机页面左右内边距固定 16px。
- Grid/Flex 子项统一使用 `min-width: 0`；长文本允许换行或按业务截断，主操作不得被挤出视口。
- 桌面两栏在手机改为按任务优先级的单列；主任务在前，支持信息在后。

## 弹窗与抽屉

### 固定宽度

| 复杂度 | 桌面宽度 | 用途 |
| --- | --- | --- |
| small | 440px | 删除、危险确认、短提示 |
| medium | 680px | 普通表单、快速记录、一般编辑 |
| large | 960px | 复杂编辑、AI 结果、并列预览 |

- 宽度实现使用 `min(规范宽度, 视口减去安全边距)`。
- 桌面视口边距至少 48px，平板至少 32px，手机至少 12px。
- 平板快速操作弹窗使用 520px；普通表单使用 680px 上限；不显示成窄手机卡片。
- large 只用于确有多区块、并列信息或复杂审批的工作流，不用大弹窗装少量内容。

### 固定结构

- panel 圆角桌面/平板固定 28px，手机 bottom sheet 顶部圆角固定 24px。
- header、body、footer 分层；桌面内边距 24px，手机 16px。
- header 标题使用 20px/700/1.35，手机 18px；描述使用 15px/1.5。
- body 是长内容唯一滚动区；header 和 footer 不随正文滚走。
- footer 操作高度桌面 44px、手机 48px，gap 10px；手机默认单列或主按钮全宽。
- backdrop 固定使用 `--overlay-backdrop (rgba(55, 40, 30, 0.28))`；不得用纯黑高透明背景或额外发光。

### 手机 bottom sheet

```css
width: 100%;
max-height: min(92dvh, var(--app-visual-viewport-height));
border-radius: 24px 24px 0 0;
padding-bottom: env(safe-area-inset-bottom, 0px);
```

- 轻量确认可以居中 modal；表单、选择器和长任务优先 bottom sheet。
- drag handle 只作提示，关闭仍需明确按钮和 Escape/返回行为。
- busy 时禁用重复提交、拖拽关闭、backdrop 关闭和 Escape 关闭。
- 错误后保留用户草稿，footer 保持可触达。

### 抽屉

- 桌面抽屉从右侧进入，宽度按 medium 或 large 任务选择，最大不超过视口。
- 手机抽屉转换为 bottom sheet 或全屏任务页，不保留狭窄侧栏形态。
- 只有跨区查看和快速编辑适合抽屉；核心长流程使用独立页面或 large modal。

## 触控与表单

- 所有交互热区最小 44px × 44px。
- 手机和平板高频按钮、输入和选择控件高度固定 48px；桌面固定 44px。
- compact 36px 只用于桌面低频工具栏，不能用于手机主操作、关闭或返回。
- 默认图标视觉尺寸 18px；关闭、返回和强调图标 20px，外层按钮仍为 44px。
- 输入圆角 14px，左右内边距 12px；textarea 最小高度 96px。
- 日期、餐次、chip 等高频选择在手机高度固定 44px；多个选择项可换行，不缩成密集小字。
- 使用真实 `button`、`input`、`select`、`textarea`；不用 div 模拟交互。

## 滚动、键盘与安全区

- sticky header、footer、composer、底部动作栏和导航必须使用项目固定 z-index token，不在业务页发明任意大 z-index。
- 手机底部区域统一加 `env(safe-area-inset-bottom, 0px)`；顶部沉浸式页面同时检查顶部安全区。
- 软键盘打开后，焦点字段、错误信息和提交按钮仍可进入视口；composer 不遮挡最后一条消息。
- modal 和 sheet 只让 body 滚动，禁止页面背景与弹层同时滚动。
- 横向滚动仅用于代码、表格或明确的可滚动选择器，不能承载页面主操作。
- loading、图片和异步内容预留稳定尺寸，避免 CLS 和滚动跳动。

## 无障碍

- overlay 使用真实 dialog 语义、标题关联、初始焦点、焦点陷阱、Escape 和关闭后焦点恢复。
- 图标按钮提供 `aria-label`；装饰图标使用 `aria-hidden`。
- 保留 `focus-visible`：2px `rgba(210, 107, 51, 0.28)` outline，2px offset。
- 状态、错误和选中不能只依赖颜色；必要时使用 live region，但避免重复播报。
- backdrop click 只能关闭允许安全退出的非 busy 弹层。
- 关闭、取消、返回的文案必须表达真实结果；不能让用户误以为草稿已保存。

## 固定验证视口

至少检查：

- 手机：375 × 812、390 × 844、430 × 932。
- 平板竖屏：768 × 1024。
- 平板横屏：1024 × 768。
- 桌面：1440 × 900。

每个任务检查长中文标题、英文/数字 ID、chip 换行、图片失败、浏览器缩放、软键盘、安全区、sticky 操作和横向溢出。报告实际查看的视口；未验证的设备明确列为缺口。
